# V3 QR Manager: System Manager Documentation

## 1. Architecture Overview: Consumer Interaction & Analytics Loop

The V3 QR Manager is designed to function not just as a static routing tool, but as a fully integrated **consumer interaction and analytics engine**. 

This document serves as the reference architecture for system managers and shop administrators to understand how physical scans translate into digital intelligence and revenue.

### 1.1 The Core Conversion Loop Design
Every QR Code generated by the V3 Platform acts as an entry point to a closed-loop analytics system. 
1. **Physical Trigger**: Consumer scans the code.
2. **Session Identification**: System mints or retrieves a long-lived, anonymized device ID and pairs it with a unique session ID. 
3. **Payload Delivery**: System delivers the configured payload (Menu, specific Campaign, or Fast Order).
4. **Behavioral Telemetry**: Actions taken on the payload (e.g., viewing items, claiming coupons) are tracked.
5. **Data Hydration**: All events are synced to the cloud DB (Supabase). If offline, they are queued strictly in `localStorage` and flushed upon reconnection.
6. **Managerial Insight**: Data populates the Shop Dashboard allowing real-time Campaign and Inventory optimization.

## 3. Managerial Capabilities Enabled by the Loop

### 3.1 Real-Time A/B Testing
Because the `ScanGateway` resolves node actions dynamically from the database, a System Manager can A/B test physical locations instantly. 
* *Example*: Change "Table 5" from `open_menu` to `open_campaign` during a slow hour without changing the physical table tent.

### 3.2 Offline-Resilient Epidemiology
The `useOfflineEventQueue` guarantees that if a shop enters a dead-zone, the engagement data is not lost. It remains tied to the consumer's device until they connect to Wi-Fi/Cellular, at which point the dashboard hydrates historically. Managers can trust the scan volumes reported in the UI.

### 3.3 Conversion Attribution
Because the Session ID carries through the entire loop (from `qr_scanned` to `reward_claimed` to `order_started`), System Managers can definitively answer: *"Did the QR code on the patio actually lead to a sale?"* The dashboard visually parses this data to show ROI per physical node.
